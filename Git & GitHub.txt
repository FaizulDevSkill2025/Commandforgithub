               Git & GitHub
===========================================
-> Version Control System
-> Keep Track of Changes.
-> Git is a tool or Software which is act as bridge between local repository and GitHub. Also Git provide command    line interface tool(CLI). Install and maintained locally.   
-> GitHub is a Cloud Storage Where Store Project Virtually for version control Service & Provide GUI Interface.

------------------------------
Instructions For Download Git
------------------------------
-> Open Chrome Browser 
-> Search By Command Git Download 
-> Then Should be come an Interface Git With Logo
-> Click this URL for open Git. 
-> After open Git interface 
-> Select by Click Windows/MacOS/Linux (According to your Windows)
-> Choose According to your windows bit (x32/x64/x86) 
-> After Download Git start install by click install option and no other option should be selected (Without need)
-> Finish Install without changed default install process. 
-> Go to Search bar of Windows and Write cmd for Command Prompt. Enter
-> Now Operate git by Command Prompt 

--------------------------
   Command For Git 
--------------------------
-> D: Enter (For Go Folder Location)
-> cd (Folder Name)
-> OR cd ../ (Back Previous Location)
-> touch note.text (Create text file in your Expected folder location)  
-> Git status - command for check file condition 
-> Git Clear - command for Clear Git interface. 
-> Git Clone - For Clone Repository. 
-> Git pwd - For Print Working Directory (Verify your exact file System Location)
-> Git add (File Name)  - For Adding Specific File in Staging Area. 
-> Git add . For all file in staging area.
-> Git add --all (For Adding all file which is untracked with in this repository)
-> Git add * - Git not add in stages deleted files.
-> Git reset- (For Reset Folder Location)
-> Git mkdir- (For Create New Folder)
-> Git ls - Use for list and inspect every file specially currently tracked in your git staging index. (From git internal prospective)
-> Git ls-files --others (For Shows Untracked Files)
-> Git ls-files --ignored (For Shows only Files being blocked by your .gitignor)
-> Git ls-files --modified  (For Shows only tracked Files that contains unsaved changes)
-> Git ls-files --unmerged (Split out a quick list of exactly which files are broken and causing the merge block)
-> Git add *.txt (For tracked only text file)
-> Git add (File Name)* - For add specific types of file. 
-> Git commit -m "Message for Last Update" - (Use this command for commit update and last changes)
-> Git log - (Use this command for Summary or check last commit)
-> Git reset HEAD~ - For reset Every thing in GitHub. 
-> Git reset --soft HEAD (For Undo Commit)
-> Git reset --soft <Commit> (Git rewinds your branch history back to that specific commit)
-> Git reset --soft HEAD~1 (Use this command for go back to the first commit)
-> Git reset --Hard HEAD~ (Use this command for go back to the Opening situation)
-> Git branch (For check current branch)
-> Git branch (branch name) (For create  new branch)
-> Git checkout (branch name) (For go to specific branch)
-> Git rm -r <Folder Name> For remove folder with file recursively. 
-> Git rm <file name> For remove specific file. 
-> Git rm <file name> -f (For remove file forcefully) 
-> Git rm -rf --cached . (This forcefully (f) and  recursively (r) delete every thing from git tracked index)
-> Git merge <branch name> (For merge with specific branch)
-> Git merge <main> -m "git message" (For merge with main branch)
-> Git push -u origin main (For push main branch)
-> Git fetch (Use for download the latest commits, branch's, and tags from the remote repository without modifying your local working files)
-> Git pull (Use For download the latest changes from a remote repository and immediately integrate them into your local working directory)
-> Git cherry-pick <commit-hash> For marge specific commit.
